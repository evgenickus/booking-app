## Варианты запуска сервиса

### 1. Локальный запуск

1.1. Установка зависимостей
  
    poetry install


    * Требуется установка только при первом запуске

1.2. Активация виртульного окружения

    poetry shell


    * Активируется перед работой с проектом

1.3. Запуск тестов

    pytest -v

1.4. Запуск приложения

    uvicorn src.app.main:app

    Адрес сервиса
      http://127.0.0.1:8000/docs
    
    * Запуск из папки booking-app/

### 2. Запуск в контейнере

2.1. Сборка образа

    docker build -t booking-app-backend .


2.2. Запуск контейнера

#### Доступен запус сервиса с тестами и без

2.2.1 Для запуска контейнера с тестами выполните команду:

    docker run --name backend-container -p 8888:8000 -v sqlite_data:/app booking-app-backend:latest tests

    Адрес сервиса
      http://127.0.0.1:8888/docs

    * Приложение запустится только после успешного выполения тестов


2.2.2 Запуск контейнера без тестов:

    docker run --name backend-container -p 8888:8000 -v sqlite_data:/app booking-app-backend:latest

    * Выполнение тестов будет пропущено и сразу запустится приложение


### 3. Запуск в docker compose

3.1 Запустить сервис c тестами:

    docker compose up
    
    Адрес сервиса
        http://127.0.0.1:8888/docs


    * Перед запуском остановите и удалите контейнер backend-container
    

## Обзор приложения


В приложении доступны следующие группы адресов по которым можно делать HTTP запросы:

 - Authenticate (Аутентификация)
 - Users (Пользователи)
 - Rooms (Комнаты)
 - Bookings (Бронирования)


![Main](./images/main.png)


### 1. Аутентификация

Эти маршруты нужны для реализации аутентификации пользователя.

GET запрос Verify User By Token проверяет валидность JWT токена

POST запрос Login For Access Token проверяет логин и пароль пользователя и выдаёт JWT токен
