Варианты запуска сервиса:

1. _________Запустить локально:____________________

   1.1. _______Установка зависимостей______________
  
    poetry install


    * Требуется установка только при первом запуске

   1.2. _______Активация виртульного окружения_____

    poetry shell


    * Активируется перед работой с проектом

   1.3. _______Запуск тестов_______________________

    pytest -v

   1.4. _______Запуск приложения___________________

    uvicorn src.app.main:app --reload

    
    * Запуск из папки booking-app/

  Адрес сервиса

    http://127.0.0.1:8000/docs#



2. _________Запустить в контейнере__________________

   2.1. _______Соборка образа_______________________

    docker build -t booking-backend .
    docker run --name backend-container -p 8888:8000 booking-backend:latest uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload (запустить контайнер)

  2.2.________Запуск контайнера_____________________

    docker start backend-container

  2.2.________Запуск тестов_________________________

    docker exec -it backend-container pytest -v (запустить тесты)

  Адрес сервиса

    http://127.0.0.1:8888/docs#


3. _________Запуск в docker compose___________

  3.1 Запустить сервисы:
    
    docker compose up -d

  3.2 Запустить тесты:

    docker compose exec backend pytest -v
    
  Адрес сервиса

    http://127.0.0.1:8888/docs#


### Обзор приложения:


В приложении доступны следующие группы адресов по которым можно делать HTTP запросы:

 - Authenticate (Аутентификация)
 - Users (Пользователи)
 - Rooms (Комнаты)
 - Bookings (Бронирования)


![Main](../images/main.png)


### 1. Аутентификация

Эти маршруты нужны для реализации аутентификации пользователя.

GET запрос Verify User By Token проверяет валидность JWT токена

POST запрос Login For Access Token проверяет логин и пароль пользователя и выдаёт JWT токен

### 2. Пользователи

### 4. Бронирования

GET запрос Read Bookings выводит список всех бронирований (активные и отмененные)
    
    http://127.0.0.1:8000/booking/all
